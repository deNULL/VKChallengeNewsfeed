== Работа на конкурс VK Mobile Challenge

Применённые решения:
* Градиент на фоне может вызывать проблемы с производительностью, поэтому на старых устройствах (до iPhone 5S) он заменяется на статичный цвет.
* Ленивая загрузка картинок в каруселях. При пролистывании ленты загружается только первая и вторая фотографии, остальные подгружаются, пока пользователь листает карусель.
* Для ускорения работы ленты вместо использования Auto Layout все элементы в ячейках выкладываются вручную в NewsfeedCell.
* Реализование поиск и подсветка найденных результатов. Подсвечивается каждое слово из поискового запроса, если оно находится в начале слова в тексте поста и от не более 3 букв до конца слова. Это позволяется не учитывать вхождения в середине слов, не прибегая к полноценному стеммингу.
* Ссылка «Показать полностью…» использует анимацию для разворачивания поста.
* Начало ввода в поисковую строку отодвигает аватарку пользователя вправо с анимацией (не было уточнено в макете).
* Если в поле для поиска есть текст, справа отображается кнопка для очистки поля. Её не было в макете, но мне показалось её наличие удобным для пользователя.
* При прокрутке ленты под статусной строкой появляется тень, прозрачность которой уменьшается по мере прокрутки.
* В тексте записей подсвечиваются и делаются кликабельными ссылки, упоминания пользователей и хэштеги. Клик по ссылке или упоминанию открывает браузер, по хэштегу — подставляет его в поле для поиска. Ссылки, не умеющающиеся на одной строке, сокращаются.
* Чтобы предотвратить слишком частые запросы к API про наборе поискового запроса, в классе API реализован debounce — запросы не отправляются, пока не прошло хотя бы 150 мс с последнего изменения запроса.

== Карусель

Условиями не оговорено, как должна себя вести карусель, если в ней фотографии с разным соотношением сторон.

Я реализовал два варианта: расчёт среднего соотношения сторон всех фото и статичная высота карусели (по умолчанию так) и динамически меняющаяся высота поста при листании карусели. Недостаток первого варианта в том, что фотографии оказываются обрезаны, если они сильно отличаются от среднего. Недостаток второго — неудобно быстро листать карусель (анимация мешает), и само изменение высоты может создавать неприятный эффект.

Для настройки используются две константы:
* NewsfeedCell.USE_DYNAMIC_CAROUSEL_HEIGHT. Если true, то высота карусели будет подстраиваться под текущего фото и анимироваться при листании. В противном случае высота будет статичной.
* NewsfeedCell.ALLOW_CAROUSEL_CLIPPING. Если true, то фотографии будут вписаны внутрь карусели с добавлением полей. В противном случае фотографии будут заполнять отведённое им место, обрезая свои края. Флаг ипользуется, только если NewsfeedCell.USE_DYNAMIC_CAROUSEL_HEIGHT = false.
